Description of the settings values.
NAME DESCRIPTION
database/feedbase The database file name for RSS feeds, including the complete path. When you read your feeds with Arado Websearch, the feed URLs, which send the news items, are saved in this database. This file is usually named feedbase.sql. The settings value shows you the path, where the feedbase.sql file is located. You can edit this database by using the NewsTicker Tab and check the "Edit" checkbox. A new table opens with the RSS/Atom Feed-URLs,
database/ipbase The detabase file name for Devices (or Peers), including the complete path. To network your devices with Arado Websearch you need to save the IP-Adresses of other devices which their users. This file is usually named ipbase.sql.
database/urlbase The database file name for all URLs of your Arado Websearch. This file is usually named urlbase.sql. The settings value shows you the path, where the urlbase is located.
engine_peersweep/maxage Each device, which connects will be deleted after the maximum age of x days. Default is 9 days. Please run Arado Websearch at least once per Week or better check once per day your RSS News Items in the Newsticker Tab or let it run constantly, so that you show to other devices, you are active. Otherwise they will remove you after the period of maxage not beeing online.
engine_peersweep/period The peerseep peroid defines how often the devices are checked, if they are outdated or not. Currently all IP-Addresses are checked each hour, if they are offline for a longer time, adjusted by the value "maxage".
engine_rss/lastpolled This value shows the Feed-Nickname, which hass been polled the last time.
engine_rss/pollperiod

Pollperoid is set to a default to poll each 30 minutes a new feed. If you have Arado Websearch contantly running, all your entered feeds will be queried. The value is seconds, so if you set the pollperiod to 60 and you have 60 feeds in your feedbase.sql, each feed is polled once per hour. Please make sure, to ask a RSS feed not more often than each 20 minutes.

engine_policy/dailycache The daiylcache saved the URLs, you just got in from other devices, and checks, if they are already given. It´s for a redundancy check, so the SQL Database has less work to look it up, if the URL is already there. The default value is 10.000 URLs. You can increase that, in case you get fast much URLs into your database.
http/address Your external IP adress is here set. Please lookup your external IP address e.g. here: www.whatismyip.com and enter the value. It works for IPv4, IPv6 and DNS entries.
http/coolingtime Coolingtime means, how long the URL synchronisation allows you CPU a pause to cool down. If you have a fast CPU, it should not need much coolingtime.
http/port The Port of your IP address. For Arado Websearch the default port 29998 is used, but as the transfer of URLs is done over the http-protocol, port 80 or 8080 is appropriate as well, especially if you run Arado Websearch as a Portal or Node on a webserver.
http/run You can set the values true and false. Please set the value to "true", if you want Arado Websearch networked.
http/getask You can set the values true and false. This flag indicates whether your arado node imports data and device (peer) IPs from other arado nodex. Please set the value to "true", if you want Arado Websearch networked.
http/getgrant You can set the values true and false. This flag indicates whether your arado node sends data in response to other arado nodes' request for data. Please set the value to "true", if you want Arado Websearch networked.
http/putallow You can set the values true and false. This flag indicates whether your arado node accepts data offered from other arado nodes. Please set the value to "true", if you want Arado Websearch networked.
http/putoffer You can set the values true and false. This flag indicates whether your arado nodes offers to send data to other arado nodes. Please set the value to "true", if you want Arado Websearch networked.
personal/elvis Elvis is the Prefix for each added device. You can rename it to any king name of your country or language. It is just a suggestion to name new devices, you have in any case the option to set the nickname of new added devices..
personal/language Language defines your ISO home language for the plugin "translate". It sets the language, in which the webpage of the URL, you are going to browse, should be translated.
personal/uuid UUID is a Universally Unique IDentifier. Each Arado Websearch Device has a unique UUID. You can change it here, but it is not recommended, as other devices then will not recognize you. You get with a new UUID a new Identity within your Arado Websearch Network.
plugins/dirpath Arado Websearch allows to plug in services. Here the path is defined, where you save the plugins. Restart Arado Websearch and the plugins are visible. There are several plugins out e.g. to convert Youtube URLs to MP3 files for free, to translate webpages or post the URL to your favourite social network like Twitter, RetroShare or Facebook, Digg etc. Plugins are third party development and found on several locations and boards on the net.
proxy/host You can run Arado Websearch over a proxy. Just set the IP-Address of the proxy here.
proxy/password Set the password of the proxy, in case the proxy needs one.
proxy/port Proxyport defines the port of the proxy.
proxy/type Proxytype defines the type of the proxy. There is http or socks5 mostly used.
proxy/useproxy Useproxy can be set to "true" or "false". Set it to true, if you want to run Arado Websearch over a proxy, you defined.
proxy/user In case the proxy reuqests a password, the account or user for this password is defined here.
rssview/advanced Advanced view of RSS can be set to "true" or "false". If you set it to "true", the Interface of the RSS-Newsticker-Tab shows two tables, one in the left for your News, one in the right for the Feed-URLs. You can un-check the Edit Box in the Tab as well, to make the right table disappear.
traffic/maxrandomA
traffix/maxrandomB
traffic/maxrandomC

Maxrandom defines, how many URLs within one transfer are included. A range of 40 - 80 URLs is appropriate for one chunk to be sent or requested. Random URLs are randomly picked out of the other devices, that means you have a good chance to get old URLs as well and backup that database.

traffic/maxrecentA
traffic/maxrecentB
traffic/maxrecentC		 Maxrecent defines, how many URLs within one transfer are included. A range of 40 - 80 URLs is appropriate for one chunk to be sent or requested. Recent URLs are the newest and freshest URLs of the other device, that means you get only the latest incmoing URLs of other devices and stay up to date.
traffic/urlFrequencyA Frequency A defines, how often an IP-Address in the Constant Devices List will be asked for recent or random URLs. As Constant Devices or A-Peers are mostly kown Friends with a good URL database, you can contact them e.g. 30 times per hour, that means each IP is asked for URLs every 2 minutes.
traffic/urlFrequencyB Frequency B defines, how often an IP-Address in the Remote Devices List will be asked for recent or random URLs. As Remote Devices or B-Peers are mostly kown Friends of Friends with a most probably good URL database, you can contact them e.g. 45 times per hour, that means each IP is asked for URLs every few minutes.
traffic/urlFrequencyC Frequency C defines, how often an IP-Address in the Flexible Devices List will be asked for recent or random URLs. As Flexible Devices or C-Peers are mostly kown as Nodes from the Galaxy with an unknown URL database, you can contact them e.g. 10 times per hour, that means each IP is asked for URLs every six minutes or set it to a slow contact freuency to avoid devices with an non-premium quality standard of URL databases.
transfer/addresses You can set the values true and false. This flag indicates whether or not your arado node exchanges device (peer) IP addresses with other arado nodes. Please set the value to "true", if you want Arado Websearch networked.
transfer/urls You can set the values true and false. This flag indicates whether or not your arado node exchanges URL data with other arado nodes. Please set the value to "true", if you want Arado Websearch networked.
urldisplay/autorefresh Autorefresh = "true" updates the search result list of URL, respective the recent incoming URLs. If you want to look up URLs in the search results without a refresh, then set it to "false".
urldisplay/maxshown Urldisplay-Maxshown sets the value, how many URLs the recent-button should show.
urldisplay/refreshperiod Refreshperoid defines the time, how fast a Refresh of the URL-Display should be done.
urldisplay/slowperiod Slowperoid defines the refresh time for the url counter.
